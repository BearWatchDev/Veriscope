About Veriscope

What is Veriscope?

Veriscope is a specialized malware analysis and detection engineering platform designed for security analysts, threat hunters, and SOC teams. It automates the tedious, time-consuming process of transforming raw malware artifacts into actionable detection rules and threat intelligence.

Traditional malware analysis workflows require analysts to manually extract strings, identify indicators of compromise, research MITRE ATT&CK techniques, and hand-craft detection rules for YARA and SIEM platforms. This process can take hours or even days per sample. Veriscope compresses this entire workflow into a single automated pipeline, delivering comprehensive analysis results in seconds.

The Challenge

Modern malware increasingly relies on sophisticated obfuscation techniques—Base64 encoding (standard, URL-safe/JWT), GZIP/zlib/bzip2 compression, hexadecimal encoding, ASCII85/Base32 variants, binary string encoding, JavaScript escapes (\xHH, \uXXXX), repeating-key XOR ciphers, ROT13/Caesar ciphers, URL encoding, PowerShell UTF-16LE obfuscation, and complex multi-layer encoding chains (often 6+ layers deep). Analysts waste valuable time manually decoding these layers before they can even begin their investigation.

Once decoded, extracting actionable IOCs (URLs, IP addresses, domains, registry keys, mutex names, file paths, cryptocurrency addresses) requires pattern matching expertise and careful validation. Mapping these behaviors to MITRE ATT&CK techniques demands deep framework knowledge. Finally, writing production-ready YARA and Sigma rules requires specialized syntax knowledge and careful testing.

Veriscope eliminates these bottlenecks by automating the entire chain: advanced multi-layer deobfuscation → IOC extraction → ATT&CK mapping → rule generation.

Core Capabilities

Advanced Multi-Layer Deobfuscation

27 specialized decoders unwrap up to 6 layers of nested obfuscation: Base64 (standard, PowerShell UTF-16LE, URL-safe), GZIP/zlib/bzip2 compression, hexadecimal (with noise filtering), string reversal (detects reversed base64), ROT13/Caesar with 45+ malware keywords, URL encoding, JSON/JavaScript extractors, character codes (decimal/hex formats), HTML entities, and multi-byte XOR (0xDEAD, 0xDEADBEEF, 0xCAFEBABE + common keys). Advanced features: SHA1 cycle detection, quality regression prevention, per-string timeout (2s), automatic plaintext detection, and comprehensive audit trails with layer-by-layer quality scoring.

Smart Deobfuscation Engine

Intelligent quality regression detection prevents over-decoding by tracking quality scores at each layer and selecting the best intermediate result. Automatic preset rotation tries multiple configurations (balanced → malware_analysis → aggressive → deepseek_optimized) when default fails. Real-time progress with Server-Sent Events displays animated decode chains. Optimized decoder ordering: Extractors → String transforms → URL → ROT13 → Base64 → Compression → Multi-byte XOR → Single-byte XOR. Configurable presets, timeouts, and XOR settings. Handles short strings, security keywords, and complex encoding chains with 80% test accuracy.

IOC Extraction

Extracts URLs, IPv4 addresses, domains, email addresses, registry keys, Windows mutex names, file paths, and cryptocurrency wallet addresses. Uses regex patterns validated against real-world malware samples. Operates on both original and decoded strings for comprehensive IOC discovery.

Entropy & Keyword Analysis

Shannon entropy calculation identifies encrypted/compressed payloads (>7.0 = likely encrypted). Quality scoring with 60+ security keywords (backdoor, credential, registry, malware, netcat) enhances detection of malicious patterns. Analyzes both original and deobfuscated content. Detects ROT13 encoding, Base64 artifacts, URL encoding remnants, and hex patterns. Weighted scoring: word frequency (45%), letter ratio (25%), structure (20%), encoding penalty (10%).

MITRE ATT&CK Mapping

Maps detected behaviors to MITRE ATT&CK techniques with confidence scoring based on match counts. Covers persistence, privilege escalation, defense evasion, credential access, discovery, lateral movement, command and control, and exfiltration tactics. Interactive modal interface groups techniques by tactic for easy review.

Interactive Analysis Dashboard

Real-time animated progress bar (0-100%) tracks deobfuscation across all strategies and presets. Method chain visualization shows successful decode sequences (e.g., "base64 → hex → rot13 → xor"). Clickable summary cards for strings, decoded content, IOCs, and ATT&CK techniques. Deobfuscation trace timeline with layer previews and quality scores. IOC breakdown by category with count badges. User-selectable findings enable custom YARA/Sigma rule generation with checkbox interface.

YARA Rule Generation

File-based detection signatures include both obfuscated and deobfuscated strings for comprehensive coverage. High-entropy strings (40+ chars) auto-added to catch encoded payloads. Generates catch-all rules and IOC-specific rules for targeted detection. Metadata includes ATT&CK technique mappings, TLP classification, author/date stamps, and sample hashes. User-selectable findings via checkbox interface enable custom rule creation with chosen IOCs only.

Sigma Rule Generation

SIEM-compatible detection rules in Sigma YAML format for log-based detections: network activity, process execution, registry modifications, file operations. Includes high-entropy encoded strings for comprehensive malware coverage. Compatible with Splunk, Elastic, QRadar, ArcSight, and 50+ SIEM platforms. Generates catch-all and IOC-specific rules with ATT&CK mappings, severity levels, and TLP classification for operational deployment.

Comprehensive Reporting

Generates structured Markdown reports with executive summaries, detailed deobfuscation traces, IOC breakdowns, ATT&CK technique mappings, and detection rule templates. Export complete analysis results in JSON format for integration with threat intelligence platforms. One-click download of individual YARA/Sigma rules or complete analysis packages for incident response documentation.

Security & Usage Guidelines

Static Analysis Only

Veriscope performs static text-based analysis exclusively. It does NOT execute binaries, run sandboxed environments, or perform dynamic analysis. All operations are read-only and safe for production analyst workstations.

Detection Rules are Templates

Generated YARA and Sigma rules are TEMPLATES designed to accelerate detection engineering. They require human review, tuning, and validation before deployment in production environments. Always test rules in a controlled environment first to avoid false positives.

ATT&CK Mapping Limitations

Technique mappings are heuristic-based and derived from keyword matching and pattern detection. They are NOT definitive behavioral analysis. Always correlate with additional threat intelligence, sandbox reports, and manual analysis before making attribution decisions.

Responsible Use

Veriscope is built for defensive security operations. Use it to analyze threats, build detections, and protect your networks. Always ensure compliance with applicable laws and regulations.
